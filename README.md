### 二、遇到的问题及解决方法

1. **问题一：语音情绪识别模型过拟合明显**

   - 现象：训练集准确率明显高于验证集，验证准确率波动较大。
   - 解决方法：
     - 降低学习率并采用分阶段训练策略；
     - 调整Batch Size提高训练稳定性；
     - 增加数据标准化处理步骤。
   - 效果：模型验证准确率稳定在合理区间，波动减小。
2. **问题二：文本情感分类结果不稳定**

   - 现象：大模型在隐性情绪表达下容易出现类别偏移。
   - 解决方法：
     - 优化Prompt结构，明确限制输出类别；
     - 增加输出格式检查与异常重试机制；
     - 引入规则加权融合语音情绪结果进行辅助校正。
   - 效果：分类稳定性明显提升。
3. **问题三：系统接口调用延迟问题**

   - 现象：网络波动时接口响应时间不稳定。
   - 解决方法：
     - 增加异常捕获与自动重试机制；
     - 优化请求数据结构，减少冗余字段。
   - 效果：系统稳定性增强，避免程序中断。

### 三、实践总结（不少于300字）

在本阶段毕业设计推进过程中，我从单一模型训练逐步过渡到完整系统集成与论文规范化撰写，整体实践能力得到显著提升。最初阶段主要关注算法本身的效果，但在系统联调过程中逐渐意识到，工程实现与算法优化同样重要。例如，在语音情绪识别模块中，模型准确率虽然是核心指标，但数据预处理规范性、输入维度统一以及训练稳定性对最终结果影响更大。通过不断调试与分析，我逐渐掌握了模型过拟合问题的排查思路，包括调整超参数、加入正则化手段等。

此外，在对话生成模中，我体会到情绪信息不仅是分类结果，更是生成策略的控制信号。如何让情绪真正参与对话生成，是本课题的重要难点。通过改进Prompt结构与情绪提示策略，使情绪标签从“辅助信息”转变为“生成条件”，系统回复的情感一致性明显增强。

论文撰写方面，通过参考优秀硕士论文结构，我进一步理解了学术论文的逻辑组织方式，从“背景—方法—实验—分析—总结”的整体框架出发，使文章更加系统化。同时，在格式规范化过程中提高了学术写作严谨性。

总体而言，本阶段不仅完成了核心技术攻坚，也提升了问题定位与系统化思考能力，为后续完善模型细节与答辩准备奠定了坚实基础。
